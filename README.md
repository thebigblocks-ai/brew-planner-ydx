# Планировщик производства

Веб-приложение для планирования производственных циклов на пивоваренном или безалкогольном производстве.

По горизонтали отображается календарь: дни, недели, месяцы и годы. По вертикали отображаются производственные емкости, ЦКТ, сгруппированные по производственным площадкам. Каждый производственный цикл отображается как полоска в строке соответствующего ЦКТ: от даты начала цикла до даты розлива готовой продукции.

## Основные возможности

### Производственные циклы

- Создание, редактирование и удаление циклов.
- Автоматический расчет даты окончания по формуле: дата начала + длительность цикла.
- Проверка пересечений: внутри одного ЦКТ циклы не могут накладываться друг на друга.
- Перемещение циклов по графику внутри ЦКТ и между ЦКТ.
- Изменение длительности цикла перетаскиванием левой или правой границы.
- Защита от случайного перемещения и изменения длительности через подтверждение.

Для цикла можно указать:

- сорт;
- ЦКТ;
- дату начала;
- длительность;
- дату окончания;
- стадию производства;
- плановый выход готовой продукции, л;
- распределение по таре;
- цвет сорта;
- комментарии.

### Стадии производства

Поддерживаются стадии:

- Брожение;
- Созревание;
- Карбонизация;
- Готово к розливу.

Цвет полоски цикла задается сортом. Цвет меняется только у плашки статуса:

- Брожение - коричневый;
- Созревание - желтый;
- Готово к розливу - зеленый.

### Режимы отображения

- Полный.
- Компакт.

Компактный режим включен по умолчанию и нужен для просмотра большого количества ЦКТ.

Масштаб календаря:

- Дни;
- Недели;
- Месяцы.

В недельном масштабе из полоски цикла убирается стадия, чтобы текст не наслаивался.

### Производственные площадки и ЦКТ

- Можно добавлять несколько производственных площадок.
- ЦКТ привязываются к конкретной площадке.
- Площадки визуально разделены на графике.
- Площадки можно сворачивать и разворачивать.
- У площадки есть карточка редактирования названия.

В карточке ЦКТ можно редактировать:

- название;
- полный объем;
- производственную площадку;
- комментарий.

ЦКТ можно переставлять местами перетаскиванием в левом списке, но только внутри своей производственной площадки. Перетаскивание ЦКТ из одной площадки в другую через список запрещено.

### Комментарии к циклам

Комментарии можно добавлять на конкретную дату внутри конкретного цикла через `Добавить -> Комментарий`.

Комментарий отображается на цикле отдельной цветной точкой. При наведении или клике можно просмотреть текст. Комментарии можно редактировать и удалять.

Шаблоны задач:

- Контроль плотности;
- Снять осадок;
- Сухое охмеление;
- Задача пюре/фруктов/соков.

### Шаблоны сортов

Приложение ведет справочник шаблонов сортов. Если сорт впервые использован при создании варки, он попадает в справочник.

В шаблоне сорта сохраняются:

- название;
- наименование по ДС;
- длительность цикла;
- плановый выход готовой продукции;
- цвет по умолчанию.

При создании новой варки можно начать вводить название сорта, выбрать его из подсказки, и приложение автоматически подставит длительность, плановый выход ГП и цвет сорта.

### Распределение по таре

В цикле есть блок распределения планового выхода ГП по таре:

- банка 0,45 л, шт;
- кега 20 л, литры.

Если указан только плановый выход ГП, распределение автоматически не считается. Если заполнить количество банок или литры в кеги, приложение рассчитает оставшийся объем во второе поле.

Количество банок и кег округляется до целого для удобства восприятия. Литры показываются справочно.

Сумма распределения по таре не может превышать плановый выход ГП. Если сумма больше планового выхода, приложение показывает предупреждение и не дает сохранить цикл.

### Ближайшие розливы

Кнопка `Ближайшие розливы` открывает список розливов.

Доступные горизонты просмотра:

- 14 дней;
- 21 день;
- 1 месяц.

В списке показаны:

- неделя;
- дата;
- сорт;
- ЦКТ;
- площадка;
- количество банок;
- количество кег.

По каждой неделе считается итог по банкам и кегам. Под кнопками горизонта показывается итог за выбранный период: литры, банки, кеги.

### Календарь розливов

Кнопка `Календарь розливов` открывает календарь для сотрудников отдела продаж.

В календаре видно, в какой день какой продукт поступает на склад и в каком количестве:

- банки;
- кеги;
- площадка.

При открытии календарь автоматически прокручивается к месяцу, в котором находится сегодняшняя дата.

### Верхняя статистика

В верхней части отображаются показатели:

- занято емкостей на сегодня;
- ГП в ЦКТ;
- синхронизация.

При клике на `ГП в ЦКТ` открывается всплывающий список сортов, которые сейчас находятся в производстве. Список разделен по площадкам и показывает сорт, ЦКТ и плановый выход ГП. По каждой площадке считается общий плановый объем ГП.

### Выделение дат

- Сегодняшний день подсвечивается прозрачной розовой заливкой по всей высоте графика.
- Клик по дате в верхней части календаря выделяет выбранный день прозрачной голубой заливкой.
- Клик в пустое место снимает выделение.

### Темы

Есть светлая и темная тема. Переключатель темы находится в верхнем блоке управления.

## Авторизация и роли

В приложении есть вход по логину и паролю.

Роли в интерфейсе:

- Администратор;
- Редактор;
- Читатель.

Технические названия ролей в базе:

- `admin` - администратор;
- `manager` - редактор;
- `reader` - читатель.

Права:

- администратор может все, в том числе добавлять пользователей;
- редактор может редактировать план, но не может добавлять пользователей;
- читатель может только просматривать данные.

В верхней левой части отображается текущий пользователь и роль.

Также отображаются пользователи, которые сейчас онлайн. Они показываются цветными квадратами с инициалами. При наведении на квадрат видно полное имя.

## Совместная работа и сохранение данных

Приложение подключено к общей базе Supabase.

Сохраняются:

- производственные площадки;
- ЦКТ;
- производственные циклы;
- комментарии;
- шаблоны сортов;
- пользователи и роли.

При включенном Realtime в Supabase изменения, сделанные одним пользователем, обновляются у других пользователей без ручного обновления страницы.

Для сохранения пользовательского порядка ЦКТ в Supabase в таблице `tanks` должно быть поле `sort_order`.

```sql
alter table public.tanks
add column if not exists sort_order integer default 0;
```

## Экспериментальный Yandex-only контур

В папке `yandex/` добавлен прототип backend без Supabase:

- `yandex/function/` - Cloud Function API;
- `yandex/api-gateway.yaml` - шаблон API Gateway;
- `yandex/README.md` - инструкция развертывания.

Новая схема:

```text
Yandex Object Storage site -> Yandex API Gateway -> Yandex Cloud Function -> Yandex Object Storage data
```

После проверки `/health` и `/auth/login` у API Gateway можно переключать `index.html` с Supabase на Yandex API.

## Верхнее управление

`Добавить`:

- Варку;
- Комментарий;
- Емкость;
- Площадку.

`Ближайшие розливы` открывает список розливов на выбранный горизонт.

`Календарь розливов` открывает календарь плановых розливов.

`Шаблоны сортов` открывает справочник сортов.

`Сегодня` перемещает календарь к сегодняшней дате.

`Тема` переключает светлую и темную тему.

## Публикация

Приложение находится в файле `index.html`.

После изменений файл нужно загрузить в репозиторий GitHub. Cloudflare Pages автоматически заберет изменения из GitHub и опубликует новую версию сайта.

Статус деплоя можно смотреть в Cloudflare Pages в разделе `Deployments`.

## Рекомендации по работе

1. Сначала создайте производственные площадки.
2. Затем добавьте ЦКТ и привяжите их к площадкам.
3. Создайте или проверьте шаблоны сортов.
4. Добавляйте производственные циклы через `Добавить -> Варку`.
5. Используйте компактный режим для общей картины загрузки.
6. Используйте полный режим, когда нужно видеть больше деталей по каждому циклу.
7. Для отдела продаж используйте блок `Календарь розливов`.
8. Для планирования розлива используйте блок `Ближайшие розливы`.
